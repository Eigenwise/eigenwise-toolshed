# Main-branch reconciliation for the TypeScript rewrite

Base: `3e337f8` (Sidequest 2.55.0). Main has shipped 2.56.0 through 2.63.0 since the rewrite forked. This map is ordered by merge risk, not commit order. “Schema-v5 ticket” means the rewrite contract's separately deferred schema-v5 work. Version-only commits do not add runtime behavior and remain orchestrator-owned release work.

The rewrite source targets below follow `docs/ts-rewrite-contract.md`: `src/lib/*.ts`, `src/bin/*.ts`, `src/hooks/*.ts`, `test/*.test.ts`, and the corresponding skill/docs sources. Generated `lib/*.js`, `bin/*.js`, and `hooks/*.js` are outputs, not editing surfaces.

## Merge checklist

1. Resolve the `ask` / `await` decision before claiming compatibility. The rewrite currently has no `cmdAsk`, `cmdAwait`, or equivalent source handlers.
2. Land schema-v5 migration work and artifact lifecycle authority together, or keep the artifact ticket blocked. The rewrite contract currently requires schema 4 and explicitly defers schema 5.
3. Port final hook identity behavior from `9e3fd8d`, not the temporary exemptions in `4285b25`.
4. Regenerate committed JavaScript only after the TS ports are complete, then run the rewrite's full compatibility suite, MCP descriptor golden check, hook tests, and schema migration tests.
